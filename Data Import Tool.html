Categorization Rules CSV Template

Categorization rules automatically assign products to categories based on product attributes. Business Manager evaluates the rules when the XML is imported.

  1. Generate template: Click "Generate CSV Template" to download the template (37 columns)
  2. Define rules: Each row = one rule with configuration and up to 10 conditions
  3. Upload & convert: Upload CSV in Step 3, enter Catalog ID, and generate XML

Rule Logic:

  • Within a row: Conditions use AND logic (all must be true)
  • Multiple rows, same category: Rules use OR logic (any rule can match)

💡 Example

Category "red-products" with 2 rules:

• Rule 1: color=Red AND size=Large

• Rule 2: brand=RedCo

Products match if Rule 1 OR Rule 2 is true

CSV Structure (37 columns):

Configuration (Columns A-G):

  • Category ID (required)
  • Category Name (optional - adds <display-name> to XML when populated)
  • Parent Category ID (optional - adds <parent> to XML when populated)
  • Locale (default: "default")
  • Site ID (optional - adds site-id attribute when populated; default: omitted)
  • Primary Category Assignment (default: false)
  • Classification Category Assignment (default: false)

Conditions (Columns H onwards):

  • Up to 10 conditions per rule
  • Each condition = 3 columns:
  • • Attribute (product attribute name)
  • • Operator (comparison type)
  • • Value (comparison value)

Supported Operators:

equals, not_equals
contains, not_contains
starts_with, ends_with
is_one_of, not_one_of
greater_than, less_than
greater_or_equal, less_or_equal
is_defined, is_not_defined

Quick Tips:

  • Leave blank: Category Name (no <display-name>), Parent Category ID (no <parent>), Locale → "default", Site ID (no site-id attribute), Primary/Classification → false
  • Boolean values: true/false, yes/no, 1/0 (case-insensitive)
  • For is_one_of/not_one_of: Use comma-separated values (e.g., "red,blue,green")
  • For is_defined/is_not_defined: Leave Value column empty

Search Refinement Definitions CSV Template

Search refinement definitions (filters) control how products can be filtered on your storefront. This tool generates a CSV template with pre-filled examples.

  1. Generate template: Click "Generate CSV Template" to download a file with example data
  2. Modify examples: The template includes 4 working examples - edit or delete them to create your refinements
  3. Add more rows: Copy example rows or add new ones following the same pattern
  4. Upload & convert: Upload the completed CSV in Step 3, enter Catalog ID, and generate XML

How Rows Work:

  • "definition" row: Defines a refinement with its basic configuration
  • "bucket" rows: Follow the definition to group values (if needed)
  • Same Refinement ID: Links bucket rows to their parent definition
  • Edit examples: The template includes complete working examples you can modify
  • Add locales: Add columns like "Display Name (de_DE)" to support more languages

💡 4 Example Types Included

Category: Simple category filter (1 row)

Brand: Attribute with no buckets (1 row)

Color: Attribute with value buckets (5 rows total)

Price: Price ranges with thresholds (5 rows total)

Column Reference (18 columns in template, expandable):

Core Columns:

  • Refinement ID: Unique identifier to group rows
  • Row Type: "definition" or "bucket"
  • Refinement Type: category, attribute, or price
  • Bucket Type: none, values, or thresholds

Configuration:

  • Attribute ID: System attribute (e.g., "refinementColor", "brand")
  • System Flag: true/false - system vs custom attribute
  • Value Set: refinement-category, search-result, or blank
  • Sort Mode: category-position, value-name, value-count
  • Sort Direction: ascending or descending
  • Unbucketed Values Mode: show-values or hide-values
  • Cutoff Threshold: Number (e.g., 5)

Display Names (expandable):

  • Template includes: default, en_US, fr_FR
  • Add more by creating "Display Name (locale)" columns
  • Example: "Display Name (de_DE)", "Display Name (ja_JP)"
  • At least "Display Name (default)" is required

Bucket Fields (bucket rows only):

  • Bucket Currency: USD, EUR, GBP, etc. (price buckets)
  • Bucket Threshold: Upper limit (price buckets)
  • Bucket Value List: Comma-separated values (e.g., "beige,cream")
  • Bucket Presentation ID: Bucket ID (e.g., "beige")

Template Includes Examples:

The CSV template includes 12 pre-filled example rows for all refinement types. Modify or delete them as needed for your data.
